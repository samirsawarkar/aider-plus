# Aider+

Aider with Persistent Memory, Project Context, and VPS-Aware Workflows.

## Why?

Aider understands your code.

But every session you still repeat:

* Project architecture
* Deployment setup
* VPS details
* Domains
* Docker configuration
* Production requirements

Aider+ solves this by adding:

### Global Knowledge

Shared across all projects.

Examples:

* VPS infrastructure
* Cloudflare configuration
* Domains
* Docker standards
* Deployment standards
* Security standards

### Project Knowledge

Powered by AI-OS.

Aider+ automatically loads:

* Project memory
* Architecture
* State
* Decisions
* Rules
* Workflows

### Dynamic Commands

Built-in workflows:

```text
/review
/production
/deploy
/docker
/security
/debug
/architecture
/context
```

### Model Profiles

Switch models instantly:

```text
/llm1
/llm2
/llm3
/llm4
```

Configured by:

```text
~/.aider-plus/models.yaml
```

### Evidence-Based Reviews

Commands are designed to:

* Use actual code
* Use actual repository context
* Use AI-OS memory
* Avoid hallucinated findings

Every claim should be backed by:

* File paths
* Code references
* Configuration evidence

## Designed For

* VPS developers
* Solo founders
* SaaS builders
* DevOps-heavy projects
* Full-stack teams

## Architecture

```text
Aider
+
Global Knowledge
+
AI-OS Project Context
+
Dynamic Commands
+
Model Profiles
```

Aider remains the editing engine.

Aider+ adds memory and workflows.

## Example

```bash
/review authentication
```

Loads:

```text
Global Knowledge
+
AI-OS Context
+
Review Command
+
User Request
```

and performs a repository-aware review.
